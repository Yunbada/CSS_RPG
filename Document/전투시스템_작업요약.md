## 📁 프리팹 구조 (매우 중요)

Player 프리팹의 계층 구조를 반드시 숙지해야 합니다. 이 구조 때문에 많은 버그가 발생했습니다.

```
Player (루트)
├── NetworkObject, InputHandle, CharacterController, PlayerMovement, PlayerCamera, PlayerAnimation
├── Animator, OwnerNetworkTransform, OwnerNetworkAnimator
│
├── PlayerCamera (자식 오브젝트)
│   └── Camera, AudioListener
│
├── PlayerModel (자식 오브젝트 - FBX 모델)
│   └── Animator, SkinnedMeshRenderer(bodymesh 등)
│
├── RPG_Systems (자식 오브젝트) ⬅️ 핵심!
│   └── PlayerState, StatSystem, InventorySystem, EquipmentSystem
│   └── InputHandle(중복), SkillController, PlayerClass, PlayerExperience
│   └── (런타임 추가) SkillSystem, CombatSystem, FighterSkillExecutor, PaladinSkillExecutor 등등
│
└── MainCanvas (UI 프리팹 인스턴스)
```

## ✅ 완료된 작업 (순차적)

### 1단계: 기반 아키텍처 및 상태이상 시스템 구축
- **CombatSystem.cs & IDamageable** 도입
  - 기존 하드코딩된 PlayerState 참조를 벗어나, `IDamageable`을 통한 공격-피격 의존성 역전 원칙(DIP) 달성.
  - 방어구 관통력(`ArmorPenetration`) 스탯 추가 및 데미지 공식 연동. (최종 데미지의 지정 %만큼 추가 데미지 발생)
- **PlayerState.cs 고도화**
  - 화상, 출혈, 독 등의 다양한 도트 뎀을 커버하는 `ApplyDoTServerRpc` 멀티 코루틴 작성 (고정 피해, 최대체력 비례, 공격력 비례 산정식 포함)
- **PlayerMovement.cs 상태이상 방해**
  - 이속 감소용 `ApplySlowClientRpc` 개설. 스킬 피격 시 이동속도 배수를 떨어뜨려 둔화 효과 달성.

### 2단계: 다중 직업 스킬 엔진 및 각성 시스템 (Paladin)
- **PaladinSkillExecutor.cs (성기사 전용 실행기)** 제작 완료
  - 1번(전진베기), 2번(축복의 방패), 3번(빛의징벌)...9번 궁극기 덱 구현.
  - 평타 타격 시 게이지가 차오르고(+5), **'축복의 방패'** 사용 시 에너지 수치에 따라 강력한 방어구 관통 장거리 투척 or 근접 접근 강타 분기 스크립트 작성.
  - 버프형 스킬(불굴의 의지)를 위해 `StatSystem.cs`에 자체 스탯 증가(`AddModifier` 등) 모디파이어 기능을 노출.
- **각성 시스템(Awakening) 개통**
  - `PlayerClass`에 `awakeningLevel` NetVar 신설.
  - `SkillSystem`에서 단축키 조작 시 `PlayerExperience.Level.Value`를 체크하여, Lv.60 달성 시 7,8번 스킬 자동 언락, Lv.90 달성 시 9번 궁극기 스킬이 자동 언락되도록 연결 완료 (각각 1차, 2차 각성 기믹).

### 3단계: 프론트엔드 UI/UX
- **직업 선택 창 확장 (C키)**: UIBuilder 로직에 성기사(Paladin, index 5) 직업군 셀렉션 UI 노출 코드 삽입 완료.
- **PlayerExperience UI 실시간 연동**: 우측 상단 중앙 지역에 `<Lv. 00> \n <Exp: 0000 / 0000>` 포맷으로 현재 서버와 연동되는 캐릭터 성장률 실시간 표기 반영.
- **인벤토리 토글 기반 인터페이스 (N키)**
  - 임시 디버그용 N키 OnGUI 패널을 완전히 지우고, N키를 누를 때마다 화면 우측에 '인벤토리 (아이템 슬롯 9개 임시 UI)' 컴포넌트가 나오게끔 `UIBuilder.cs`, `UIGameHUD.cs`를 리팩토링.
  - 인벤토리가 열려있는 도중에는 스킬 시전이 전면 차단되며, 1번을 누를 시 강제로 100레벨 반영, 2,3번 지정 시 1차, 2차 각성이 임의로 돌파되는 더미 개발용 테스트 로직을 `SkillSystem.cs`에 주입 완료.

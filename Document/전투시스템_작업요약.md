# CSS_RPG 전투 시스템 작업 요약

> **최종 업데이트:** 2026-03-29  
> **프로젝트:** CSS_RPG (Unity Netcode for GameObjects 멀티플레이어 RPG)

---

## 📁 프리팹 구조 (매우 중요)

Player 프리팹의 계층 구조를 반드시 숙지해야 합니다. 이 구조 때문에 많은 버그가 발생했습니다.

```
Player (루트)
├── NetworkObject, InputHandle, CharacterController, PlayerMovement, PlayerCamera, PlayerAnimation
├── Animator, OwnerNetworkTransform, OwnerNetworkAnimator
│
├── PlayerCamera (자식 오브젝트)
│   └── Camera, AudioListener
│
├── PlayerModel (자식 오브젝트 - FBX 모델)
│   └── Animator, SkinnedMeshRenderer(bodymesh 등)
│
├── RPG_Systems (자식 오브젝트) ⬅️ 핵심!
│   └── PlayerState, StatSystem, InventorySystem, EquipmentSystem
│   └── InputHandle(중복), SkillController, PlayerClass, PlayerExperience
│   └── (런타임 추가) SkillSystem, CombatSystem, FighterSkillExecutor
│
└── MainCanvas (UI 프리팹 인스턴스)
```

### ⚠️ 중요한 주의사항
- **PlayerState가 루트가 아닌 RPG_Systems 자식에 있음**
- CharacterController(Collider)는 **루트**에 있음
- 따라서 `GetComponentInParent<PlayerState>()`로는 충돌체에서 PlayerState를 찾을 수 없음
- 반드시 `CombatSystem.FindPlayerState()` 헬퍼를 사용해야 함

## ✅ 완료된 작업 (순차적)

### 1단계: 전투 시스템 코어 구축
- **CombatSystem.cs** 생성 (MonoBehaviour)
  - NetworkBehaviour → MonoBehaviour로 변환 (런타임 AddComponent 호환)
  - `InitializeCombatSystem()`을 통한 명시적 초기화 패턴
  - 데미지 공식: Attack, Defense, CritRate, CritDamage 기반
  - 마우스 좌클릭 기본공격 구현
  - `SendDamage()` → `PlayerState.AttackTargetServerRpc()` 위임

### 2단계: 무투가 9개 스킬 구현
- **FighterSkillExecutor.cs** 생성
  - 1번 정권찌르기: 전방 단일 타격
  - 2번 승천권: 전방 타격 + 적과 함께 2m 상승 (KnockUp)
  - 3번 백스텝: 후방 2m 회피 이동
  - 4번 파쇄권: 전방 2m 대시 + 도착 시 주변 범위 타격
  - 5번 풍각: 회전 타격 + 전방 3m 전진 (1.5초)
  - 6번 연파: 전방 연속 타격 (1.5초)
  - 7번 번개춤: 주변 3m 이동 타격 (2초)
  - 8번 연탄: 장풍 연사
  - 9번 진공난무: 주변 흡인 + 강타 (2초)

### 3단계: 스킬 시스템 연동
- **SkillSystem.cs** 수정
  - 스킬 사용 중 다른 스킬 차단 (`IsUsingSkill` 플래그)
  - 모든 스킬 쿨타임 1초 통일
  - 팀 변경 시 좀비/인간 스킬 자동 전환

### 4단계: 이동 시스템 연동
- **PlayerMovement.cs** 수정
  - 스킬 사용 중 일반 이동 차단
  - `ApplyForcedMovement()` 추가 (넉업/넉백 코루틴)
  - `ResetGravity()` 추가 (백스텝/파쇄권 중력 초기화)
  - `isForcedMoving` 플래그로 강제이동 중 입력 차단

### 5단계: 네트워크 동기화
- **PlayerState.cs** 수정
  - `AttackTargetServerRpc()`: 서버 권한 데미지 적용
  - `KnockUpServerRpc()` / `KnockUpClientRpc()`: 넉업 동기화
  - `SetInvincibleServerRpc()`: 무적 상태 동기화
  - `TakeDamage()`: 최소 데미지 1 보장 (`Mathf.Max(1, amount)`)

### 6단계: 버그 수정 (핵심)

#### 6-1. 데미지 미적용 버그 (근본 원인 해결)
- **원인:** `PlayerState`가 RPG_Systems 자식에 있는데, Collider(CharacterController)는 루트에 있어서 `GetComponentInParent<PlayerState>()`가 항상 null 반환
- **해결:** `CombatSystem.FindPlayerState()` 헬퍼 메서드 추가
  - 부모 방향 + 루트→자식 방향 양방향 검색
  - CombatSystem과 FighterSkillExecutor 양쪽 모두 적용

#### 6-2. 좀비가 인간 스킬 사용 버그
- **원인:** `ExecuteSkillAttack`에서 팀 체크 없이 직업만 확인
- **해결:** `playerState.currentTeam.Value == Team.Human` 조건 추가

#### 6-3. 카메라/CharacterController 참조 누락
- **원인:** `GetComponent`로 같은 오브젝트에서만 찾아서 null
- **해결:** `GetComponentInParent`로 루트까지 올라가서 검색

#### 6-4. 백스텝/파쇄권 중력 누적
- **원인:** 스킬 중 PlayerMovement 정지 시 velocity.y가 누적
- **해결:** 스킬 시작 시 `ResetGravity()` 호출

#### 6-5. 타격 판정 개선
- **변경:** `Physics.Raycast` → `Physics.SphereCastAll` (반경 1m)
- 얇은 레이 대신 두꺼운 구체 투사로 관대한 판정
- 모든 공격 범위 +1m 증가
